# Eval Specifications

Eval specs define built-in evaluator profiles (case-level and report-level).

## Required Fields

- `id`
- `version`
- `name`
- `description`
- `category`
- `evaluator_type`
- `pydantic_class`
- `output_kind`

## Common Fields

- `cost_tier`
- `latency`
- `requires`
- `default_config`
- `source`

## Notes

- Evals are referenced from agent specs and runtime workflows.
- Prefer explicit `id:version` references.
- Benchmark datasets are specified separately under `benchmarks/`.
